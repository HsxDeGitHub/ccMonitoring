# CC Monitor 设计文档

## 背景

用户在 macOS 终端中通过 `claude` 命令使用 Claude Code。切换到其他程序工作时，需要频繁切回终端查看 Claude Code 是否仍在运行、是否在等待确认。目标是开发一个桌面悬浮监控工具，一眼就能看到所有 Claude Code 实例的运行状态。

## 需求摘要

- 支持多实例监控（用户可能同时开多个终端运行 claude）
- 四种状态显示：运行中（绿色）、等待确认（黄色闪烁）、已完成（灰色）、出错（红色）
- Python 实现，悬浮窗口始终置顶
- 按工作目录 + 终端标题区分实例
- 支持收缩到屏幕侧边 + 点击展开

---

## 架构

```
┌─────────────────────────────────────┐
│           Overlay Window            │  ← tkinter 悬浮窗口
│   展示所有 Claude Code 实例状态       │
└──────────────┬──────────────────────┘
               │ 状态列表
┌──────────────▼──────────────────────┐
│         State Engine                │  ← 进程数据 → 状态判定
└──────────────┬──────────────────────┘
               │ 原始进程数据
┌──────────────▼──────────────────────┐
│        Process Monitor              │  ← psutil 扫描 claude 进程
└──────────────┬──────────────────────┘
               │
┌──────────────▼──────────────────────┐
│         Main Controller             │  ← 1-2s 轮询循环
└─────────────────────────────────────┘
```

四个模块职责：

- **Main Controller**：启动轮询循环，协调各模块，管理窗口生命周期
- **Process Monitor**：通过 `psutil` 扫描系统中所有 `claude` 相关 Node.js 进程
- **State Engine**：根据进程数据（CPU 使用率、存活状态、退出码）判定实例状态
- **Overlay Window**：tkinter 实现的置顶悬浮窗口，展开/收缩两态

---

## 状态判定逻辑

| 状态 | 条件 | 颜色 |
|------|------|------|
| 运行中 | 进程存在 + CPU 使用率 > 5% | 绿色 |
| 等待确认 | 进程存在 + CPU 使用率 ≤ 5% | 黄色闪烁 |
| 已完成 | 进程不存在 + 退出码 = 0 | 灰色 |
| 出错 | 进程不存在 + 退出码 ≠ 0 | 红色 |

实例唯一标识：PID + 工作目录 + 启动时间。进程退出后保留 30 秒后清理（让用户有时间看到完成/出错状态）。进程识别方式：扫描 Node.js 进程，匹配命令行中包含 `claude` 关键词的进程。

## 窗口设计

### 展开状态

```
┌──────────────────────────────┐
│  CC Monitor          [−][×]  │
├──────────────────────────────┤
│  🟢 /project/api (api)      │
│  🟡 /project/web (web)      │
│  ⚫ /tmp/test     (test)     │
│  🔴 /project/db  (db)       │
└──────────────────────────────┘
```

- 每行：状态圆点 + 工作目录 + 终端标题
- 窗口置顶、半透明背景、无原生标题栏
- `−` 收缩、`×` 退出
- 可拖拽移动，支持多实例滚动

### 收缩状态

```
┌──┐
│🟡│
│  │ ← 屏幕侧边吸附的标签条
│◀▶│
│  │
└──┘
```

- 标签条吸附在屏幕左/右边缘
- 显示当前最需要关注的状态色（红 > 黄 > 绿 > 灰）
- 点击标签条展开，支持拖拽切换左右吸附

---

## 技术选型

- Python 3 + `psutil`（进程监控） + `tkinter`（GUI，Python 内置）
- 窗口置顶：`root.attributes('-topmost', True)`
- 无边框：`root.overrideredirect(True)`
- 拖拽：绑定 `<Button-1>` / `<B1-Motion>` 事件

## 文件结构

```
ccMonitoring/
├── monitor.py            # 主入口
├── process_scanner.py    # 进程扫描
├── state_engine.py       # 状态判定
├── overlay_window.py     # 悬浮窗口
└── requirements.txt      # psutil
```

## 非功能需求

- 轮询间隔 1-2 秒，CPU 占用极低
- 窗口始终置顶，不抢焦点
- 退出后不残留进程
